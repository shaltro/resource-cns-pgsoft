# resource-cns-pgsoft

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态资源的形式存放，不依赖于特定域名或后端服务，通过 GitHub Pages 可直接访问。

仓库名中的 `resource` 表示资源归档，`cns` 与 `pgsoft` 为内部项目标识，无特殊含义。

## 目录说明

```
.
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
├── index.html      # 仓库首页（可选）
└── README.md       # 本文件
```

- `pages/` 下每个文件为一个独立页面，文件名即页面路径标识。
- `assets/` 目录按需组织，建议按页面或功能模块建立子目录，便于维护。
- 根目录下的 `index.html` 可作为导航或说明页，非必须。

## 页面归档说明

- 所有页面均为纯静态 HTML，不包含后端逻辑。
- 页面之间相互独立，无依赖关系，可单独访问。
- 页面内容以信息展示、工具或演示类为主，不涉及交互式数据提交。
- 归档页面可能随时间更新或移除，不保证所有页面长期可用。

## 维护说明

- 新增页面请放入 `pages/` 目录，并确保引用的资源路径正确。
- 页面命名建议使用英文或拼音，避免特殊字符。
- 如需更新已有页面，直接覆盖文件并提交即可。
- 删除页面时，请同步清理不再使用的资源文件。
- 本仓库不设置 CI/CD，更新后 GitHub Pages 会自动部署。

## 使用方式

1. 克隆仓库到本地。
2. 在 `pages/` 目录下添加或修改 HTML 文件。
3. 提交并推送至 GitHub。
4. 通过 GitHub Pages 功能访问对应页面。

## 许可

本仓库仅用于归档目的，不对页面内容作任何形式的保证。如需引用或分发，请自行判断。
